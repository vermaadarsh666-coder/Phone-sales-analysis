# Phone-sales-analysis


## Short Description
A data analysis project that uses a Kaggle mobile phone dataset to explore phone specifications, brand trends, operating systems, 5G availability, battery capacity, camera features, and storage patterns using SQL and Power BI.

## Introduction
This project is a phone data analysis dashboard built to study the mobile market from multiple angles. The dataset was taken from Kaggle, cleaned and explored using SQL, and then visualized in Power BI for interactive reporting.

The main goal of this project is to turn raw mobile phone data into meaningful business insights that can help understand brand performance, feature distribution, and customer-facing specifications.

## Dataset / Inputs
- Source: Kaggle dataset
- Data type: Mobile phone product/specification data
- Key fields may include:
  - Brand
  - Phone name
  - Operating system
  - Battery capacity
  - Selfie camera
  - Internal storage
  - 5G availability
  - Other phone specifications



## Tech Stack
- SQL
- Power BI
- Kaggle Dataset
- Data Cleaning and Analysis

## Usage
1. Load the Kaggle dataset into your SQL environment.
2. Run the SQL queries used for cleaning, filtering, and analysis.
3. Import the processed data into Power BI.
4. Open the dashboard to explore visual insights by brand, OS, battery, camera, storage, and 5G support.

## Analysis / Features
- Brand-wise phone analysis
- Operating system distribution
- 5G availability comparison
- Battery capacity analysis
- Selfie camera feature analysis
- Internal storage distribution
- Mobile count and brand-level overview

## Results / Insights
- The dashboard shows the distribution of mobile phones across major brands.
- Android appears to dominate the operating system category.
- 5G availability can be compared between supported and non-supported devices.
- Battery capacity, storage, and camera features are visualized to identify common phone specifications.
- The report provides a clear summary of the mobile market using interactive charts and KPIs.
